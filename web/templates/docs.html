Dokumentasi API

Panduan integrasi Whatsar ke project kamu — kirim notif WhatsApp, terima balasan via webhook.

Mulai Cepat Autentikasi Format Response Session Kirim Pesan Webhook Contoh Kode Tips

Mulai Cepat — Notif WhatsApp

Alur paling umum buat project notifikasi (Laravel, Node, Python, dll.):

  1. Pairing session — buat session di Admin UI, scan QR WhatsApp.
  2. Ambil session_id — dari dashboard atau GET /api/v1/sessions.
  3. Simpan API key — set WHATSAR_API_KEY di .env server Whatsar.
  4. Kirim notif — POST /api/v1/messages/send dari backend project kamu.
  5. (Opsional) Webhook — daftarkan URL project kamu buat terima pesan masuk / balasan user.
Base URL: {{.BaseURL}} {{if .PublicURLOverride}} — dari WHATSAR_PUBLIC_URL di .env. {{else}} — otomatis dari domain yang kamu pakai buka halaman ini (mis. https://whatsar.dev saat deploy). {{end}}

Autentikasi

Semua endpoint /api/v1/* butuh API key. Kirim lewat header (disarankan):

X-API-Key: your-api-key-here

API key bisa di-generate atau diubah dari Pengaturan — tersimpan di database, tidak perlu edit .env manual.

Alternatif via query string: ?api_key=your-api-key-here (kurang aman, hindari di production).

Endpoint /health tidak butuh API key — cocok buat monitoring uptime.

Format Response

Semua response API berbentuk JSON envelope:

RESPONSE sukses & error
{
  "success": true,
  "data": { ... },
  "meta": { "request_id": "uuid" }
}

{
  "success": false,
  "error": { "code": "SEND_FAILED", "message": "..." },
  "meta": { "request_id": "uuid" }
}

Rate limit: 60 request/menit per API key.

Session

Satu session = satu nomor WhatsApp yang ter-pair. Status penting sebelum kirim pesan:

StatusArti
connectedSiap kirim pesan ✅
qr_readyMenunggu scan QR
connectingSedang menghubungkan
failedGagal — buat session baru
GET /health
Cek server hidup. Tanpa API key.
POST /api/v1/sessions

Buat session baru (untuk pairing QR).

curl -X POST {{.BaseURL}}/api/v1/sessions \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "notif-toko"}'
GET /api/v1/sessions
List semua session + status connected.
GET /api/v1/sessions/{id}
Detail satu session.
GET /api/v1/sessions/{id}/status

Cek cepat sebelum kirim notif.

{
  "success": true,
  "data": {
    "id": "uuid",
    "status": "connected",
    "connected": true,
    "phone": "6281234567890"
  }
}
GET /api/v1/sessions/{id}/qr

Ambil QR code (JSON). Field image_base64 = PNG siap tampil.

{
  "success": true,
  "data": {
    "code": "2@...",
    "image_base64": "iVBOR...",
    "status": "qr_ready"
  }
}
DELETE /api/v1/sessions/{id}
Hapus session + logout device.

Webhook — Terima Pesan Masuk

Kalau user balas notif kamu, Whatsar akan POST ke URL project kamu.

POST /api/v1/webhooks
curl -X POST {{.BaseURL}}/api/v1/webhooks \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://project-kamu.com/api/wa-webhook",
    "session_id": "abd4a712-8333-4a92-8eb1-bd922e72e47a",
    "events": ["message.in"],
    "secret": "webhook-secret-random"
  }'

session_id kosong = webhook global (semua session).

Payload yang dikirim ke URL kamu

POST https://project-kamu.com/api/wa-webhook
Content-Type: application/json
X-Whatsar-Signature: sha256=abc123...   // jika secret diisi
User-Agent: Whatsar-Webhook/1.0

{
  "event": "message.in",
  "session_id": "uuid",
  "timestamp": "2026-06-16T10:00:00Z",
  "data": {
    "from": "6289876543210",
    "chat": "6289876543210@s.whatsapp.net",
    "body": "Oke kak, siap",
    "message_id": "3EB0...",
    "type": "text",
    "is_group": false
  }
}

Verifikasi signature (PHP)

$payload = file_get_contents('php://input');
$secret  = 'webhook-secret-random';
$sig     = $_SERVER['HTTP_X_WHATSAR_SIGNATURE'] ?? '';
$expected = 'sha256=' . hash_hmac('sha256', $payload, $secret);

if (!hash_equals($expected, $sig)) {
    http_response_code(401);
    exit('invalid signature');
}

$data = json_decode($payload, true);
// proses $data['data']['body'], $data['data']['from'], dll.

Contoh Integrasi

PHP / Laravel

// app/Services/WhatsAppNotif.php
public function kirim(string $nomor, string $pesan): array
{
    $response = Http::withHeaders([
        'X-API-Key' => config('services.whatsar.key'),
    ])->post(config('services.whatsar.url') . '/api/v1/messages/send', [
        'session_id' => config('services.whatsar.session_id'),
        'to'         => $nomor,
        'text'       => $pesan,
    ]);

    return $response->json();
}

// .env project kamu:
// WHATSAR_URL={{.BaseURL}}
// WHATSAR_KEY=your-api-key-dari-pengaturan
// WHATSAR_SESSION_ID=uuid-dari-dashboard

Node.js

async function kirimNotif(to, text) {
  const res = await fetch(`${process.env.WHATSAR_URL}/api/v1/messages/send`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': process.env.WHATSAR_KEY,
    },
    body: JSON.stringify({
      session_id: process.env.WHATSAR_SESSION_ID,
      to,
      text,
    }),
  });
  return res.json();
}

Python

import os, requests

def kirim_notif(nomor: str, pesan: str):
    r = requests.post(
        f"{os.environ['WHATSAR_URL']}/api/v1/messages/send",
        headers={"X-API-Key": os.environ["WHATSAR_KEY"]},
        json={
            "session_id": os.environ["WHATSAR_SESSION_ID"],
            "to": nomor,
            "text": pesan,
        },
        timeout=15,
    )
    r.raise_for_status()
    return r.json()

Tips Production

  • Nomor format internasional — Indonesia: 62xxx tanpa 0 di depan (bukan 08xxx).
  • Cek status dulu — panggil /sessions/{id}/status sebelum blast notif; handle kalau connected: false.
  • Jangan hardcode API key — simpan di .env project, jangan commit ke git.
  • Deploy via tunnel — Cloudflare Tunnel / reverse proxy supaya project cloud bisa akses Whatsar di VPS/STB.
  • Anti-spam — rate limit 60 req/menit; untuk bulk notif, queue di sisi project kamu.
  • Session terputus? — cek dashboard; biasanya auto-reconnect. Kalau gagal, scan QR ulang.
